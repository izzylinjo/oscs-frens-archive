## What this is
Automated YouTube Shorts bot that pulls top Twitch clips from OSCS and their
friend group, runs them through a two-step Discord review, then posts to YouTube
Shorts on a staggered schedule.

## Pipeline summary
1. APScheduler triggers Twitch fetch at noon and midnight
2. twitch.py fetches top 5 clips per streamer since last_checked_at
3. Clips inserted into DB with status = fetched
4. discord_bot.py posts new clips to #clip-review as embeds
5. React ✅ = approved, ❌ = rejected, reply = custom_title set + approved
6. Drop Twitch clip URLs into #clip-inbox for manual submission
7. queue.py fill_queue(bot) picks up approved clips every 30 mins
8. For each: yt-dlp download → Whisper transcription → Gemini 3 titles → ffmpeg overlay
9. Clip posted to #clip-final-approval with title options + 1️⃣2️⃣3️⃣❌ reactions
10. React 1️⃣2️⃣3️⃣ picks title → status = queued, reply = custom title → queued
11. post_next_queued() picks oldest queued_at clip, uploads to YouTube
12. overlay file deleted after successful upload
13. on_ready recovers any awaiting_title clips if bot was restarted mid-flow

## Clip state lifecycle
fetched → approved → awaiting_title → queued → uploaded
              ↓              ↓
           rejected       rejected

## Key decisions made
- No vertical crop (9:16) for now — post landscape, revisit month 2
- No AI video trimming — Twitch clips are pre-trimmed
- Streamer credit: small bottom-left overlay + full socials in YouTube description
- Titles: descriptive style, focus on personality/moment not game, under 60 chars
  Custom title reply in Discord is the main override — Gemini is a starting point
- Rejected clips: archived in DB, never posted
- YouTube Shorts only at launch — TikTok/Instagram/Twitter later
- MAX_QUEUE_SIZE = 36 (approx 18 hours of content)
- Approved clips that exceed queue cap stay as approved, not lost
- Only queue.py sets status to queued — update_status enforces queued_at
- File paths are NOT stored in DB — derived from naming convention in queue.py
  Convention: downloads/{clip_id}_overlay.mp4
  If file missing on post attempt, clip reverts to approved for reprocessing
- Whisper model: "small" locally (GPU), plan to upgrade to "large" on Oracle VPS
- Gemini model: gemini-3.1-flash-lite-preview — 500 RPD free tier (gemini-2.0-flash
  had zero quota allocation on this Google Cloud project)
- yt-dlp for downloads: Twitch CDN URL trick (_mp4_url) returns thumbnail JPEGs now,
  yt-dlp is the reliable path
