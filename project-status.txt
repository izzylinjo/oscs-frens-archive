# OSCS & Frens Archive — Project Status
Last updated: session 3 (full pipeline code complete, YouTube setup pending)

## What this is
Automated YouTube Shorts bot that pulls top Twitch clips from OSCS and their
friend group, runs them through a Discord review step, then posts to YouTube
Shorts on a staggered schedule. Weekly and monthly compilations go to the main
channel as long-form videos.

## Pipeline summary
1. APScheduler triggers Twitch fetch at noon and midnight
2. twitch.py fetches top 5 clips per streamer since last_checked_at
3. Clips inserted into DB with status = fetched
4. discord_bot.py posts new clips to #clip-review as embeds
5. You react ✅ = approved, ❌ = rejected, reply = custom_title set + approved
6. You can drop Twitch clip URLs into #clip-inbox for manual submission
7. queue.py fill_queue() picks up approved clips every 30 mins
8. For each: downloads mp4 → Whisper transcription → Gemini title → ffmpeg overlay
9. File path flows through functions as return values — nothing stored in DB
10. update_status('queued') sets queued_at automatically
11. queue.py post_next_queued() picks oldest by queued_at, calls youtube.upload()
12. overlay file deleted after successful upload
13. All approved clips archived in SQLite permanently
14. End of week: ffmpeg compiles approved clips → you approve → long-form post
15. End of month: monthly recap compiled from archive

## Key decisions made
- No vertical crop (9:16) for now — post landscape, revisit month 2
- No AI video trimming — Twitch clips are pre-trimmed
- No sound effect edits
- Streamer credit: small bottom-left overlay + full socials in description
- Streamer name in title only if it improves it (not forced)
- Rejected clips: archived in DB, never posted
- YouTube Shorts only at launch — TikTok/Instagram/Twitter later
- Weekly + monthly compilations = long-form on main channel
- MAX_QUEUE_SIZE = 36 (approx 18 hours of content)
- Approved clips that exceed queue cap stay as approved, not lost
- Only queue.py sets status to queued — update_status enforces queued_at
- Status lifecycle: fetched → approved → queued → uploaded (or rejected)
- Discord channel not DMs for review (scales better, easier to debug)
- last_checked_at updated to max(clip.created_at) not now() to prevent gaps
- custom_title is the authoritative field (set by Discord reply)
  title is the display/working field (overwritten by Gemini or custom_title)
- File paths are NOT stored in DB — derived from naming convention in queue.py
  Convention: downloads/{clip_id}_overlay.mp4
  If file missing on post attempt, clip reverts to approved for reprocessing
